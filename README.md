> [커밋 타입] 리스트
> 
> feat : 기능 (새로운 기능)
> 
> fix : 버그 (버그 수정) 
> 
> design : CSS 등 사용자 UI 디자인 변경 
> 
> refactor : 리팩토링
> 
> style : 스타일 (코드 형식, 세미콜론 추가: 비즈니스 로직에 변경 없음)
> 
> docs : 문서 (문서 추가, 수정, 삭제)
> 
> test : 테스트 (테스트 코드 추가, 수정, 삭제: 비즈니스 로직에 변경 없음)
> 
> chore : 기타 변경사항 (빌드 스크립트 수정 등)
> 
> rename : 파일 혹은 폴더명을 수정하거나 옮기는 작업만 하는 경우
> 
> remove : 파일을 삭제하는 작업만 수행한 경우
